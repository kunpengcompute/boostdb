# MySQL特性之间的兼容性

## 特性之间的兼容性<a name="ZH-CN_TOPIC_0000001874354741"></a>

NVMe SSD原子写特性与其他特性不涉及兼容性问题，而其他MySQL特性之间存在一定的兼容性关系。

- NVMe SSD原子写特性是硬件层面的优化，该特性与其他MySQL特性不涉及兼容性问题。
- 其他特性需要考虑与其他MySQL特性之间的兼容性问题。特性之间的兼容性说明如[**表 1** 特性之间的兼容性说明](#特性之间的兼容性说明)所示。

**表 1** 特性之间的兼容性说明<a id="特性之间的兼容性说明"></a>

|特性名称|兼容性说明|
|--|--|
|MySQL网络多路径优化|与其他特性兼容。|
|MySQL KAEZstd页压缩解压缩优化|与其他特性兼容。|
|鲲鹏GCC CFGO反馈优化|与其他特性兼容。|
|Gazelle网络优化|与其他特性兼容。|
|CRC32指令优化|与其他特性兼容。|
|MySQL细粒度锁优化|与其他特性兼容。细粒度锁优化特性是无锁优化特性的前置，必须先应用细粒度锁优化特性才能应用无锁优化特性。|
|MySQL无锁优化|与其他特性兼容。细粒度锁优化特性是无锁优化特性的前置，必须先应用细粒度锁优化特性才能应用无锁优化特性。|
|NUMA调度优化|与MySQL并行查询优化、MySQL可插拔在线向量化分析引擎不兼容，与其他特性兼容。如果undo_spaces_lock优化特性和NUMA调度优化特性共用，则需要先应用NUMA调度优化特性，再应用undo_spaces_lock优化特性。|
|hash_table_locks优化|与其他特性兼容。|
|undo_spaces_lock优化|与其他特性兼容。如果undo_spaces_lock优化特性和NUMA调度优化特性共用，则需要先应用NUMA调度优化特性，再应用undo_spaces_lock优化特性。|
|线程计数器优化|与其他特性兼容。|
|MySQL可插拔在线向量化分析引擎|与MySQL并行查询优化不兼容，与其他特性兼容。|
|MySQL并行查询优化|与MySQL可插拔在线向量化分析引擎优化不兼容，与其他特性兼容。|
|MySQL计算路径优化|与其他特性兼容。|
|MySQL Binlog优化|与其他特性兼容。|
